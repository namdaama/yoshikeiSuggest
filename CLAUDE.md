# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## リポジトリの性格

このリポジトリには **ソースコードは存在しない**。ヨシケイ開発株式会社向けの営業提案（自社Web注文サイト「楽楽web」の改修・改善パートナー提案）に関する資料一式を管理する場所である。したがってビルド・テスト・Lint等の開発コマンドは存在しない。作業は資料（テキスト / PPTX / PDF / XLSX）の読み取り・更新・新規作成が中心になる。

収録物：

- `about` — 提案の背景・目的・課題・To-Be・基本方針を記した地の文。提案書や派生資料を作る際の**一次ソース**として最優先で参照する。
- `ヨシケイ開発株式会社  御中注文ページ改修のご提案.pptx` — 先方提出用の提案スライド（ファイル名の「御中」と「注文」の間は全角スペース2つ。リネーム時は要注意）。
- `楽楽web_バックエンド処理ロジック一覧.pdf` — ベンダー提供用の**抽象版**処理カテゴリ整理（1. 注文データ受信・正規化 〜 8. 管理・運用向け参照 の8カテゴリ）。実装・技術構成・アルゴリズム・業務固有詳細は意図的に含めていない。抽象度を下げない。
- `楽楽web_改善効果_測定構造_算出定義付き.xlsx` — 改善効果の測定構造と算出定義。KPI議論の根拠資料。

## 提案のコアメッセージ（ぶれさせないこと）

`about` に書かれた方針は提案全体の前提であり、派生資料でこれと矛盾する記述を作ってはならない：

- **目的は中長期成果**：短期の売上最大化ではなく、注文数・顧客単価・継続率を行動/関係性改善で底上げする。
- **現基盤は活用する**：フルリプレイス/フルスクラッチは対象外。
- **役割は"改善推進パートナー"**：受託ベンダーの置き換えではない。企画・判断の主導権を外部が補完し、軽微改修を短サイクルで回す体制を作る。
- **PDCAを回せる状態を作る**が中心テーマ：「要件確定→見積→実装→納品」から「仮説→小改修→数値確認→次」へ移行させる。
- **NG方針**（`about` 末尾）：フルスクラッチ／従来型丸投げ／EC構築実績のみでのベンダー選定 — これらに寄る提案は禁止。

## 取り扱い上の注意

- 公開資料の言語は **日本語**。ファイル更新・新規作成時も特段の指示がない限り日本語で書く。
- 楽楽webのバックエンド処理詳細は先方から抽象レベルでしか開示されていない（PDFの注記参照）。**具体実装・技術スタック・アルゴリズムを推測して資料に書き込まない**。書く場合は「想定」「仮説」等を明示する。
- 先方固有の業務データ・内部仕様を創作して書かない。`about` とPDF/XLSXから読み取れる範囲に留める。
- ファイル名に日本語と全角スペースを含むものがある。シェルで扱う際はクオートし、リネームを伴う操作は事前に確認する。
- バイナリ資料（PPTX/XLSX）は差分が追いにくい。編集前にユーザーへ意図を確認し、編集後は何を変えたかを要約して伝える。
