# 運営向けダッシュボード 用語集

> **用途：** ダッシュボード（[dashboard_mockup.html](dashboard_mockup.html)）および提案書（[dashboard_proposal.md](dashboard_proposal.md)）で使用する指標名・分析用語・記号・略号の定義集。運営担当者が読み下す際の参照資料。
>
> **位置付け：** 先方既定義の 3層6KPI は `楽楽web_改善効果_測定構造_算出定義付き.xlsx` を一次ソースとする。補完指標・分析用語は業界標準に準拠。

---

## 1. 指標（先方既定義 6 KPI）

### 第1層：行動KPI（測定頻度：週次）

| KPI | 算出式 | 見方 |
|---|---|---|
| **ログイン後直帰率** | ログイン後1ページのみ閲覧して離脱したユーザー数 ÷ ログインユーザー数 | 高止まり時は、TOP画面や次アクションの提示に課題あり。注文有無を問わず測定する。 |
| **注文未実施週の再訪率** | 翌週にログインした未注文会員数 ÷ 当該週に注文しなかった会員数 | 改善により数ポイント上昇すれば有効。自然変動があるため前週比で判断。 |

### 第2層：関係KPI（測定頻度：月次）

| KPI | 算出式 | 見方 |
|---|---|---|
| **月1回以上閲覧会員比率** | 当月1回以上ログインした会員数 ÷ 全有効会員数 | 生活の中でサービスが忘れられていないか。低下傾向は関係断絶のリスク。退会せず放置層の可視化に有効。 |
| **未注文週→翌週復帰率** | 翌週に注文した元未注文会員数 ÷ 当週未注文会員数 | 一時的未利用からの回復力を見る。UI改善・人フォローの効果が出やすい指標。季節性を考慮して比較。 |

### 第3層：結果KPI（測定頻度：四半期）

| KPI | 算出式 | 見方 |
|---|---|---|
| **平均注文頻度** | 期間内の総注文回数 ÷ 注文を行った会員数 | 長期的に利用頻度が改善しているか。第1・2層改善の結果として確認。短期では判断しない。 |
| **実質離脱数** | 一定期間（例：8週）ログインも注文もない会員数 | 自然消滅を含む離脱傾向の把握。解約手続きの有無は問わない。減少していれば改善が効いている。 |

## 2. 補完指標（業界標準）

BtoC食品宅配／EC向け運営ダッシュボードで一般的に使用される指標。先方6KPIの読み解きを補助する位置付け。

| 指標 | 算出式 | 用途 |
|---|---|---|
| **平均注文額** | 期間内の売上金額 ÷ 期間内の注文件数 | 単価傾向の把握 |
| **月次アクティブ会員数** | 当月に1回以上ログインした会員数 | サービス規模の推移 |
| **顧客生涯価値（推計）** | 平均注文額 × 平均注文頻度 × 平均継続期間（簡易推計） | 顧客価値の長期推移 |
| **ファネル** | 訪問 → ログイン → カート追加 → 注文完了の遷移率。各段階の会員数／前段階の会員数 | 離脱が集中している段階の特定 |
| **リテンション・コホート** | 加入月ごとに会員をグループ化し、経過月数別に継続率を見る。加入月Mの会員のうち、Nヶ月後も継続している会員数 ÷ 加入月Mの会員数 | どの加入期の会員が定着しているか |
| **構成比** | 部分の値 ÷ 全体の値 | コース別の注文寄与度等 |

## 3. 分析用語（数値の読み方）

| 用語 | 意味 |
|---|---|
| **pt（ポイント）** | 比率（%）の差を表す単位。「42.1% → 44.2%」の変化は「+2.1 pt」と表記。％ そのものの変化率（+5%）と区別するために使う。 |
| **前週比 / 前月比 / 前Q比** | 対象期間の値と直前期間の値との比較。変化率（%）または差分（件数・pt）で示す。 |
| **前週同曜日比** | 前週の同曜日（例：今週の月曜 vs 先週の月曜）との比較。曜日効果を除いた日次推移の把握に使用。 |
| **4週移動平均** | 直近4週間の平均値。単週のブレを吸収してトレンドを見るための平滑化手法。週次チャートで本系列と重ねて表示。 |
| **閾値（しきいち）超過アラート** | あらかじめ定めた基準（例：前週比 −5% 以下）を超えた項目を自動的に警告として表示する仕組み。 |
| **着地予測** | 締切時点での最終件数・金額の見込み。現時点の進捗と前週同時点比から推計。 |
| **施策リリース日注釈** | UI改修・施策投入の実施日をチャート上に垂直線として表示する手法。KPIの転換点と施策の因果関係を読み解くため。 |
| **分子／分母** | 比率計算の構成要素。算出式の透明性確保のため各KPI表示に併記する場合がある。 |

## 5. チャート・UI 用語

| 用語 | 説明 |
|---|---|
| **KPIカード** | 単一指標を現値・前期比・スパークラインでまとめた1枚の小パネル。ダッシュボード最上段に配置。 |
| **スパークライン** | KPIカード内の軸ラベルを省いた小さな折れ線グラフ。値の傾向（上昇／下降／横ばい）を直感的に示す。 |
| **横棒ランキング** | 数値の大きい順に項目を並べた水平バーチャート。FC別実績・コース別構成等に使用。 |
| **2軸折れ線** | 左右に異なるY軸を持つ折れ線グラフ。スケールの異なる2系列を同時に比較する際に使用。 |
| **棒グラフ＋参考線** | 当期の値を棒で、比較対象（前年同期等）を点線で重ねる表示。四半期トレンドに使用。 |
| **ヒートマップ** | セルの色濃淡で値を表現する格子状の図。コホート継続率等に使用。濃緑＝高値／赤橙＝低値の2極スケール。 |
| **前週位置ゴースト線** | ファネルで前週の位置を点線で重ねる表示。前週比の変化を視覚的に示す。 |
| **Callout（コールアウト）** | チャート近傍に配置する強調通知ブロック。Signal／Alert／Note の3種を用途で使い分け。 |
| **Signal** | 施策検討のヒントとなる兆候（例：ファネル特定段階の低下）。オレンジ系で表示。 |
| **Alert** | 閾値超過・要即対応の警告（例：FC別の急落）。赤系で表示。 |
| **Note** | 補足情報・前提条件等の注釈。ニュートラル色で表示。 |

## 6. 時間・頻度の表記

| 表記 | 意味 |
|---|---|
| **日次** | 毎日（前日実績ベース、朝07:00配信想定） |
| **週次** | 毎週（月曜〜日曜を1週間、翌月曜朝配信想定） |
| **月次** | 毎月（月初第1営業日配信想定） |
| **四半期** | 3ヶ月単位（四半期末配信想定） |
| **W17 等** | ISO 週番号（例：W17 = 2026年 第17週 ＝ 4/20〜4/26） |
| **24Q1 / 25Q3 等** | 年次＋四半期表記（Q1=1〜3月、Q2=4〜6月、Q3=7〜9月、Q4=10〜12月） |

## 7. 記号

| 記号 | 意味 |
|---|---|
| **▲ / ▼** | 数値の上昇（▲）／下降（▼）を示すアイコン。方向の意味は指標ごとに異なる点に注意（ログイン後直帰率なら▼が改善方向） |
| **⚑** | 警告マーカー（閾値超過等） |
| **§ + 番号** | ダッシュボード内のセクション番号（例：§03 = 第3セクション） |
| **FC=XXX** | フランチャイズコード（例：FC=YKH 横浜／FC=OSK 大阪／FC=TOY 豊橋） |

## 8. 用語の読み方補足

- **改善方向とアイコンの対応：** 指標によっては "低い方が良い" ものがある。以下の指標は **数値が下がる = 改善** として表示。
  - ログイン後直帰率：▼ −1.8 pt（下がる = 改善）
  - 実質離脱数：▼ −3.1 %（下がる = 改善）
- **「自然変動」の意味：** 季節要因・曜日要因・外部要因（連休等）による短期的な揺らぎ。単週のブレを判断材料にしない根拠として用いる。移動平均で吸収する。
- **「仮説→小改修→数値確認→次」のサイクル：** 本ダッシュボードの設計思想の背景。大規模開発を経ず、小さな改修を短サイクルで回すための数値基盤として位置付けている（詳細は提案書本体参照）。
