# 川平さんへの対価交渉メッセージ叩き（3パターン）

> **取扱注意：** 対価合意前の社内下書き。送信前に片岡さん本人の口調で調整すること。

---

## パターン1：ソフトめ

### メッセージ本文
```
川平くん、例のpptxの保守パート追記の件、ちょっと先に相談させてほしいことがあって。

今回の追記、普通に手を動かす分には全然いいんだけど、ちょうどいいタイミングな気がするので、本件全体の進め方をいっかい整理できたらなと思って。

というのも、保守パートって今回の提案の中で一番重いところだし、もし受注になったら僕の関わり方もそれなりに大きくなりそうで。そのへんの体制とか、僕がどこまで持つのか／持ちきれないところをどう切るのか、川平くんと握っておいた方がお互い動きやすいかなと。

商流的には、リバプレで受けてもらって僕はそこから業務委託、みたいな形が一番シンプルかなと想像してるんだけど、どう思う？受注に至らなかった場合の今回の作業分の扱いも含めて、ざっくりでいいから一度話せたら嬉しい。

急ぎじゃないので、都合いいタイミングで15〜30分もらえたら！
```

### 意図
友人関係を損なわず、「相談」という柔らかい入口から本質的な論点（商流・関与範囲・今回作業の扱い）に自然に触れる。対価という言葉を直接使わず「体制」「進め方」に包むことで、川平さんが身構えずに応じられる空気を作る。

### リスク
婉曲すぎて「ただの相談」と受け取られ、肝心の対価や契約形態の議論が先送りになる可能性。作業を先に進められてしまい、未払い前提で関わり続ける流れが固定化するおそれ。

### 使うべきシーン
関係性がまだ温まりきっていない、または川平さんが忙しそうで心理的負荷を上げたくないとき。初回の切り出しとして軽くジャブを打ち、次の打ち合わせで詰める二段構えが取れる場合。

## パターン2：中間（推奨想定）

### メッセージ本文
```
川平くん、pptxの保守パート追記の件、着手する前に一点すり合わせさせて。

今回の作業、実は対価とか契約の話を僕らまだしてなくて、僕の方でも切り出すタイミング逃してたごめん。着手前の今がちょうどいいと思うので、本件まわりの進め方を一緒に整理できたら嬉しい。

論点として僕が思ってるのは3つで、
①今回の追記作業そのものの扱い（業務委託として見積出す／提案活動として一旦持ち出す、など）
②受注できた場合に僕がどこまで関わるか（企画・実装・保守運用のどこまで、月額＋スポットみたいな形がありうるか）
③受注に至らなかった場合の今回作業分の扱い

商流としては、リバプレで元請けしてもらって僕に業務委託、が一番現実的かなと思ってて、自分から提案したい。特に保守は全量を個人で抱えるのは難しいので、どこを僕が持ってどこをリバプレ側／別リソースで持つかの線引きも一緒に考えたい。

近々30分くらい話せるとありがたい！
```

### 意図
論点を明示しつつ「自分も切り出しが遅れた」と対等に非を引き取ることで、詰め寄るトーンを回避。商流(b)を自分から出すことで主導権を持ちつつ、保守の線引きも「一緒に考える」フレームで投げ、川平さんを共同設計者の位置に置く。

### リスク
論点が3つ並ぶことで「きちんとした話」感が出て、川平さんによっては構えさせるかも。また論点を自分から整理してしまうことで、相手の選択肢が自分の設定した枠内に収束する（交渉の初手を自分で切る）形になる。

### 使うべきシーン
関係性は良好で、かつ早めに合意形成したいとき。作業着手前で、ここで決めておかないと後で揉める論点がはっきりしている今の状況。デフォルトで推奨。

## パターン3：フラット・論点提示型

### メッセージ本文
```
川平くん、pptx保守パート追記の件、着手前に決めておきたいこと共有させて。

前提：僕はこれ個人で受ける想定。現時点で対価・契約形態・受注後の関与範囲は未合意。

論点：
1. 今回の追記作業の扱い：業務委託として見積を出すか、提案活動扱いで持ち出すか。
2. 受注時の僕の関与範囲と報酬：企画／実装／保守運用のどこまで担当するか。月額固定＋スポットの組み合わせを想定。
3. 不受注時の今回作業分の扱い。
4. 保守の分担：全量を個人で持つのは現実的でないので、僕が持つ範囲とそれ以外の持ち方を切り分けたい。

商流の提案：リバプレ元請け→僕に業務委託、の形を想定。異論あれば教えて。

上記について、今週中に30分もらえると助かる。候補日時投げてくれれば合わせます。
```

### 意図
感情的な緩衝材を削ぎ、論点と提案を箇条書きで明示して合意形成を最短距離で進める。文書として後から参照できる形にすることで「言った／言わない」を防ぎ、川平さん側にも判断材料を一度に渡す。

### リスク
友人同士のチャットとしては硬く、「急に business モードになった」と受け取られる可能性。川平さんが心理的に身構え、返信までのラグや防御的な反応を招くおそれ。peer-to-peerの気軽さから外れる。

### 使うべきシーン
すでに何度か口頭で触れていて論点が共有されている状態で、文字に落として確定させたいとき。または川平さん自身が論点整理型のコミュニケーションを好むタイプだと分かっているとき。メールに近い重さなので、チャットで使うなら前後に軽い会話を添えるとよい。

---

## 共通設計

どのパターンでも、最後に「30分くらい話そう」と口頭に寄せる出口を置いてある。チャットで全部決めきろうとせず、詰めは通話に委ねる構造にすることで、ニュアンスの行き違いを減らせる。
